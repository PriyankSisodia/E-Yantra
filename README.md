<p>The above dictionary conveys the following information:</p>
<p><strong>Total number of balls to be dispensed by BD:</strong> There are in total five balls that will be dispensed by the BD.
<strong>Color of each ball:</strong> There will be two red, one green and two blue balls dispensed by the BD in random order.
<strong>Drop location of each ball:</strong> The drop location i.e., the name of Collection Box in which balls are to be dropped, is given as list of strings corresponding to the color of each ball.</p>
<blockquote>
<p><strong>NOTE:</strong></p>
<ul>

<li>This means that at the very beginning, the first ball dispensed by the <strong>BD</strong> may or may not be <strong>red</strong> in color.</li>
<li>Similarly, at the very end, the last ball dispensed by the <strong>BD</strong> may or may not be <strong>blue</strong> in color.</li>
</ul>
</blockquote>
<h4><a id="user-content-goal" class="anchor" aria-hidden="true" href="#goal"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Goal</h4>
<p>There will be pre-defined number of balls and their drop locations that will be dispensed by the <strong>BD</strong> at <strong>regular interval</strong> of <strong>80 simulation seconds</strong>, which can be determined from the <strong><code>ball_details.json</code></strong> file provided before the theme run.</p>
<ul>
<li>Before the theme run starts, teams will import their <strong>Ball Balance Platform</strong> design in <strong><code>arena_scene.ttt</code></strong> file. The coordinates to which these platforms are to be positioned will be defined in <strong><code>arena_scene_details.pdf</code></strong>.</li>
<li>Teams will start their Python client script, process the four maze  image files provided using Image Processing and generate these mazes on  top of the <strong>four</strong> Platform Tables in the <strong><code>arena_scene.ttt</code></strong>. The maze generation takes place with the help of customisation Lua script(s) in CoppeliaSim.</li>
<li>Once the simulation starts, the Ball Dispenser (<strong>BD</strong>) will start dispensing each ball at regular interval of <strong>80 simulation seconds</strong>.</li>
<li>The <strong>Theme Run Timer</strong> will start <strong>as soon as the first ball</strong> is transferred on to Table-4 (<strong>T4</strong>) by Conveyor-Belt-4 (<strong>B4</strong>).</li>
<li><strong>T4</strong> has to navigate the ball through maze on it and pass the ball to the next table i.e., <strong>T1</strong> or <strong>T2</strong> or <strong>T3</strong> based on the designated drop location mentioned in <strong><code>ball_details.json</code></strong> file.</li>
<li>The next table will then navigate the ball through maze on it and drop the ball in the appropriate Collection Box (<strong>CB</strong>).</li>
<li>The second ball will be dispensed at <strong>80 simulation seconds</strong> after the first ball was dispensed. This will repeat itself until the last ball is dispensed.</li>
<li>The <strong>Theme Run Timer</strong> will <strong>end after 480 simulation seconds</strong> from the start of timer.</li>
<li>The objective is to drop all the balls to their respective <strong>CB</strong> within <strong>480 simulation seconds</strong>.</li>
</ul>

<h2><a id="user-content-theme-rules" class="anchor" aria-hidden="true" href="#theme-rules"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Theme Rules</h2>
<ul>
<li>
<p>The <strong>maximum time</strong> allotted to complete the task is <strong>480 simulation seconds</strong>.</p>
</li>
<li>
<p>A <strong>maximum of two runs</strong> will be given to each team from which the <strong>best of the two runs</strong> will be counted as the <strong>final score</strong>.</p>
</li>
<li>
<p>A <strong>maximum of one restart</strong> is allowed per run wherein the Python client script, the CoppeliaSim simulation and the Theme Run  timer will start again. In case a team requires a <em>second restart</em>, the <strong>run will be ended</strong> and <strong>maximum time</strong> for the task (<strong>480 simulation seconds</strong>) will be considered for that run.</p>
</li>
<li>
<p>If during simulation, any kind of collision or balancing of Platform Tables causes any <strong>changes in the arena configuration</strong>, then it will result in <strong>immediate end</strong> of the run and <strong>maximum time</strong> for the task (<strong>480 simulation seconds</strong>) will be considered for that run.</p>
</li>
<li>
<p>Teams will be provided with the following files before the start of theme run:</p>
<ul>
<li><strong><code>arena_scene.ttt</code></strong></li>
<li><strong><code>arena_scene_details.pdf</code></strong></li>
<li><strong><code>ball_details.json</code></strong></li>
<li><strong><code>maze_t1.jpg</code></strong>, <strong><code>maze_t2.jpg</code></strong>, <strong><code>maze_t3.jpg</code></strong> and <strong><code>maze_t4.jpg</code></strong></li>
</ul>
</li>
<li>
<p>Teams will import their <strong>Ball Balance Platform</strong> design in <strong><code>arena_scene.ttt</code></strong> file. The coordinates to which these platforms are to be positioned will be defined in <strong><code>arena_scene_details.pdf</code></strong>.</p>
</li>
<li>
<p>Teams will start their Python client script, process the four maze  image files provided using Image Processing and generate these mazes on  top of the <strong>four</strong> Platform Tables in the <strong><code>arena_scene.ttt</code></strong>. The maze generation takes place with the help of customisation Lua script(s) in CoppeliaSim.</p>
</li>
<li>
<p>Once the simulation starts, the Ball Dispenser (<strong>BD</strong>) will start dispensing each ball at regular interval of <strong>80 simulation seconds</strong>.</p>
</li>
<li>
<p>The <strong>Theme Run Timer</strong> will start <strong>as soon as the first ball</strong> is transferred on to Table-4 (<strong>T4</strong>) by Conveyor-Belt-4 (<strong>B4</strong>).</p>
</li>
<li>
<p>Teams have to make <strong>T4</strong> navigate the ball through maze on it and pass the ball to the next table i.e., <strong>T1</strong> or <strong>T2</strong> or <strong>T3</strong> based on the designated drop location mentioned in <strong><code>ball_details.json</code></strong> file.</p>
</li>
<li>
<p>The next table will then navigate the ball through maze on it and drop the ball in the appropriate Collection Box (<strong>CB</strong>).</p>
</li>
<li>
<p>The <strong>second ball</strong> will be dispensed at <strong>80 simulation seconds</strong> after the first ball was dispensed.</p>
</li>
<li>
<p>This process will repeat itself until the last ball is dispensed.</p>
</li>
<li>
<p>The <strong>Theme Run Timer</strong> will <strong>end after 480 simulation seconds</strong> from the start of timer.</p>
</li>
<li>
<p>The objective is to drop all the balls to their respective <strong>CB</strong> within <strong>480 simulation seconds</strong>.</p>
</li>
<li>
<p>Each <strong>Platform Table</strong> that is able to <strong>pass / drop each ball to the next table / the designated CB within the stipulated amount of time</strong> is considered as <em><strong>OK Tested!</strong></em></p>
</li>
<li>
<p>A run ends and the</p>
<p>Theme Run Timer</p>
<p>is stopped when:</p>
<ul>
<li>If the maximum time limit of <strong>480 simulation seconds</strong> for completing the task is reached <strong>OR</strong></li>
<li>If the team needs a second restart but has already used first restart for that run <strong>OR</strong></li>
<li>If any of the Platform Tables or any kind of collision while performing the task causes any <strong>changes in the arena configuration</strong>.</li>
</ul>
</li>
<li>
<p>Second run will start after the first whilst resetting the score, timer and the arena.</p>
</li>
<li>
<p>The score of both runs will be recorded and best of two runs will be considered as the team's final score.</p>

</li>
</ul>
